● MODULE 01
01

ปฐมบท: รู้จัก Blender สำหรับ MG

ติดตั้งโปรแกรม จัด workspace ให้เหมาะกับงาน Motion Graphics และทำความเข้าใจ navigation จนเป็นธรรมชาติ — ขั้นตอนที่หลายคนข้าม แต่กำหนดความเร็วในงานทั้งหมดของคุณ

4 บท1.2h เวลาBeginner
01

ติดตั้ง Blender 4.x และตั้งค่าระบบ

— เริ่มจากศูนย์ ใช้เวอร์ชันที่ stable ที่สุด

ดาวน์โหลด Blender จาก blender.org เลือก LTS (Long-term Support) สำหรับงานจริงเสมอ จากนั้นเข้าไปที่ Edit → Preferences เพื่อตั้งค่าก่อนเริ่ม

  • System tab → เลือก GPU (CUDA/OptiX สำหรับ NVIDIA, HIP สำหรับ AMD, Metal สำหรับ Mac)
  • Save & Load → เปิด Auto Save ตั้งเวลา 2 นาที กันงานหายเวลา crash
  • Themes → เลือก "Blender Dark" หรือ custom เพื่อลดอาการตาล้า
  • Input → เปิด Emulate 3 Button Mouse ถ้าใช้ trackpad
PRO TIP · 01

เปิด Developer Extras ใน Interface tab ทันที — มันจะเปิดของหลายอย่าง เช่น copy data path สำหรับ drivers ในภายหลัง

02

เข้าใจ Interface — รู้จักทุก editor ในเวลา 10 นาที

— UI ของ Blender ไม่ยากถ้ารู้จัก editor

หน้าจอ Blender แบ่งเป็น Areas ที่สามารถเปลี่ยนเป็น editor อะไรก็ได้ ลากมุมเพื่อแบ่ง drag-and-drop มุมเข้ากันเพื่อรวม สำหรับ MG เราต้องใช้ editor หลัก 6 ตัว

  • 3D Viewport — สนามทำงานหลัก ทุกอย่างเริ่มและจบที่นี่
  • Outliner — สมุดรายชื่อ object ทั้งหมดในฉาก
  • Properties — กำหนดค่า render, world, object, modifier
  • Timeline — ดู keyframe ในแกนเวลา
  • Graph Editor — ปรับ curve ของทุก animation (สำคัญที่สุด!)
  • Shader Editor — ทำ material แบบ node
# Workspace สำหรับ Motion Graphics (แนะนำ) LEFT → 3D Viewport (ใหญ่) RIGHT → Outliner + Properties (แบ่งครึ่ง) BOTTOM→ Timeline / Graph Editor (สลับด้วย Ctrl+Tab)
03

Navigation: ปุ่มลัดที่ต้องจำให้ได้ภายในวันแรก

— ความเร็วในการทำงานเริ่มจากนิ้วของคุณ

ลืมเรื่องเลื่อนเมาส์ Blender ออกแบบมาให้ใช้คีย์บอร์ดเป็นหลัก ฝึกปุ่มเหล่านี้จนกล้ามเนื้อจำได้

G เคลื่อนที่ R หมุน S scale G+X เคลื่อนแกน X R+Z+90 หมุน Z 90°
Numpad 1/3/7 Front/Right/Top view Numpad 5 Ortho/Persp Numpad 0 Camera view . (period) โฟกัสที่ object
04

ตั้งค่าโปรเจคใหม่ — Scene units, Frame rate, Output

— ตั้งค่าให้ถูกตั้งแต่ต้น ประหยัดเวลาภายหลัง

ก่อนสร้าง object ตัวแรก ตั้งค่า scene ก่อนเสมอ ขั้นตอนนี้ใช้เวลาแค่ 1 นาที แต่ป้องกันปัญหาในอนาคต

  • Output Properties → Resolution 1920×1080 หรือ 3840×2160 (แล้วแต่งาน)
  • Frame Rate → 24 fps สำหรับ cinematic, 30 fps สำหรับ social, 60 fps สำหรับ smooth motion
  • Scene Properties → Units → Metric, scale 1.0 (ตั้งให้ถูกถ้าจะ import asset จากโปรแกรมอื่น)
  • Color Management → View Transform = AgX (ใช้แทน Filmic ใน 4.x — สีสวยกว่ามาก)
● MODULE 02
02

หัวใจของ MG: Keyframe Animation

เข้าใจกลไก keyframe ตั้งแต่หัวใจ จนถึงเทคนิคที่ studio ใช้จริง — รู้ว่าเมื่อไรควรใช้ Auto Keying และเมื่อไรไม่ควร

4 บท2.0hBeginner+
05

Keyframe คืออะไร — แนวคิดก่อนปุ่มลัด

— ภาพนิ่งที่บอกว่า ณ frame นี้ object อยู่ตรงไหน

คิดง่ายๆ: keyframe คือการ์ดถ่ายรูปฉากๆ หนึ่ง เราถ่าย 2 รูป Blender จะวาดเฟรมที่อยู่ระหว่างกลางให้เอง เรียกว่า interpolation งาน MG ทั้งหมด = ศิลปะของการเลือกเฟรมไหนเป็น keyframe และ curve ระหว่างกลางเป็นยังไง

I เพิ่ม keyframe (insert) Alt+I ลบ keyframe I → Location เลือก property
06

Auto Keying — ดาบสองคม

— เปิดเมื่อต้องการ ปิดทันทีเมื่อจบ

กดปุ่มกล้องสีแดงข้างปุ่ม play ใน Timeline เพื่อเปิด Auto Keying ทุกครั้งที่คุณขยับ object Blender จะใส่ keyframe ให้อัตโนมัติ — สะดวก แต่ถ้าเผลอจะพังงาน

เตือน

Auto Keying เปิดไว้แล้ว scrub timeline โดยที่ object ยังเลือกอยู่ — ถ้าเผลอกดอะไรนิดเดียว keyframe ใหม่จะแทรกเข้าไปทันที ปิดเมื่อไม่ได้ใช้เสมอ

07

Keying Sets — เก็บคีย์เฉพาะที่อยากเก็บ

— เลือก property ที่จะ key ก่อนกด I

ที่ Timeline header เลือก Keying Set เป็น LocRotScale, Location, หรือ Available เพื่อกำหนดว่ากด I แล้วจะ key อะไรบ้าง สำหรับ MG ส่วนใหญ่ใช้ Available — เก็บเฉพาะ property ที่เคย key ไว้แล้ว ลด noise ใน Graph Editor

  • LocRotScale — เก็บทั้ง position, rotation, scale (เริ่มต้น)
  • Available — เก็บเฉพาะที่มี keyframe อยู่แล้ว (มือโปรชอบ)
  • Whole Character — สำหรับ rig (ใช้ใน character animation)
08

Workflow: animation แรกของคุณ — Bouncing Cube

— โปรเจคเล็กที่ครอบคลุมทุกอย่าง

ฝึกผ่าน exercise: ทำ cube กระเด้งจากบนลงล่าง 3 ครั้งใน 60 frames ลำดับขั้น

  • Frame 1: ตำแหน่ง Z = 5, กด I → Location
  • Frame 20: ตำแหน่ง Z = 0, กด I (เด้งครั้งแรก)
  • Frame 35: Z = 2.5, กด I (กระเด้งกลับขึ้น)
  • Frame 50: Z = 0, กด I (เด้งครั้งที่ 2)
  • Frame 60: Z = 1, กด I (จบ)
  • เปิด Graph Editor → เลือก keyframe ทั้งหมด → กด T → Bezier (curve นุ่ม)
● MODULE 03
03

Graph Editor & ศิลปะของ Easing

ที่นี่คือที่ที่ animation ดิบๆ กลายเป็น cinematic จับ curve ให้อยู่หมัด เข้าใจ 12 หลัก animation ที่ Disney ใช้มาตั้งแต่ปี 1981

3 บท1.8hIntermediate
09

F-Curves — ภาษาของ animation

— ทุก animation คือกราฟ y = f(t)

เปิด Graph Editor (Shift+F6 หรือเลือกจากมุม) จะเห็นเส้นโค้งแสดง property ตามแกนเวลา แกน X = frame, แกน Y = value handle ของแต่ละ keyframe ควบคุมความ "นุ่ม" ของ transition

V เปลี่ยน handle type T interpolation mode Shift+E extrapolation mode Ctrl+B เลือก range
10

Easing — เส้นแบ่งระหว่างมือใหม่กับมือโปร

— ของจริงไม่เคลื่อนที่เร็วคงที่

Linear easing = ของในวิดีโอเกมยุค 80s ของจริงมี momentum ทุกครั้งที่ขยับ ใช้ Ease In เริ่มช้า, Ease Out จบช้า, Ease In-Out ทั้งสองข้าง สำหรับ MG สมัยใหม่ลองใช้

  • Cubic / Sine — สำหรับ camera move, fade เนียน
  • Quint / Expo — สำหรับ snap แรงๆ มี punch
  • Back / Elastic — overshoot ที่ทำให้รู้สึก playful
  • Bounce — ใช้น้อยๆ ระวัง cliché
เคล็ดลับมือโปร

กฎทอง: เร็วในตอนต้น ช้าในตอนจบ — ใช้ Ease Out (Quint หรือ Expo) เป็นค่าเริ่มต้นสำหรับ 80% ของ animation ของคุณ

11

12 หลัก Animation — Squash & Stretch, Anticipation, Follow Through

— จาก The Illusion of Life สู่ MG ปี 2026

หลักของ Disney ใช้ได้กับทุก motion ไม่ใช่แค่ตัวการ์ตูน 4 หลักที่ใช้บ่อยที่สุดใน MG

  • Squash & Stretch — บีบ-ยืดให้รู้สึกมีน้ำหนักและความเร็ว (ลูกบอลกระเด้ง)
  • Anticipation — ขยับสวนทางก่อนเคลื่อนที่จริง (ก่อนกระโดดต้องย่อ)
  • Follow Through — ส่วนปลายตามทีหลัง (หางม้า, ผม, ผ้า)
  • Slow In, Slow Out — easing นั่นเอง
● MODULE 04
04

Drivers & Constraints

เปลี่ยนงาน manual เป็น automation — ผูก property หนึ่งกับอีกตัว ใช้สมการ Python ทำสิ่งที่ keyframe ทำไม่ได้

3 บท1.6hIntermediate
12

Constraint System — ลูกศรที่บอกว่า "ตามฉันไป"

— ผูก object ตัวหนึ่งให้ขึ้นกับอีกตัว

ใน Properties panel ที่ tab รูปโซ่ คือ Object Constraints constraint สำคัญสำหรับ MG

  • Copy Location/Rotation/Scale — ทำตามอีกตัว 100% หรือบางแกน
  • Track To — หันหน้าหา target ตลอด (ใช้กับกล้อง spotlight)
  • Follow Path — เดินตาม curve ที่วาดไว้
  • Limit Location/Rotation — กำหนดขอบเขตการเคลื่อนที่
  • Child Of — parent แบบมี keyframe ได้ (เปลี่ยน parent กลางทาง)
13

Drivers — คณิตศาสตร์ที่ขับเคลื่อน animation

— ผูก property ด้วยสมการ ไม่ใช่ keyframe

คลิกขวาที่ property ใดๆ → Add Driver ค่าที่ป้อนจะมาจากสมการ ไม่ใช่ keyframe ตัวอย่างคลาสสิค: ทำให้ล้อรถหมุนตามระยะทาง

# Example: ล้อหมุนตามตำแหน่ง X ของรถ Property: Wheel.rotation_euler[1] Driver: var = Car.location.x Expression: -var / radius # Example: scale ตามระยะห่างจาก empty Property: Object.scale Driver: distance(Object, Empty) Expression: 1 + var * 0.1
14

Empty + Driver = ระบบ controller มืออาชีพ

— ทำ rig MG ใน 5 นาที

สร้าง Empty หนึ่งตัว ผูก property ของ object 10 ตัวให้ขึ้นกับ Z location ของ Empty ขยับ Empty ตัวเดียว → ทุกอย่างเปลี่ยนพร้อมกัน เป็นวิธีที่ studio ใช้ control complex animation

SUPER PRO TIP

ใช้ Custom Property บน Empty (สร้าง slider 0–1) แล้ว driver ทุกอย่างให้ขึ้นกับมัน — ได้ controller พร้อมส่งให้ลูกค้าเล่นเอง

● MODULE 05
05

Geometry Nodes สำหรับ MG

อาวุธลับของ MG ยุคใหม่ — สร้าง animation ที่ keyframe ทำไม่ได้ ทุกอย่างเป็น procedural แก้ไขได้ตลอด

5 บท3.0hAdvanced
15

Geometry Nodes คืออะไร — ทำไมเปลี่ยนวงการ MG

— programming แต่ใช้ node ไม่ใช้ code

ก่อน GN ถ้าต้องการให้ object 1000 ตัวกระจายเป็นรูปทรงเฉพาะ ต้อง keyframe ทีละตัว ทำกี่ชั่วโมงก็ไม่จบ ด้วย GN เขียน "สูตร" ครั้งเดียว ใส่ค่าเปลี่ยนได้ตลอด — เปลี่ยน 1000 เป็น 10000 ในวินาทีเดียว

16

Distribute Points + Instance — first principles

— สร้างป่าจาก mesh ตัวเดียว

วาง modifier Geometry Nodes บน plane → เปิด Geometry Nodes editor → workflow มาตรฐาน

Group Input (mesh) ↓ Distribute Points on Faces (density: 50) ↓ Instance on Points ↑ instance: Group Input → Object Info (tree.blend) ↓ Rotate Instances (random Z 0–360°) ↓ Group Output

ผลลัพธ์: ป่า 50 ต้นที่กระจายแบบสุ่ม หมุนคนละทิศ — ใช้เวลา 30 วินาที

17

Time-based animation ใน GN — ใช้ Scene Time

— animation ที่ไม่ต้อง keyframe

Add → Input → Scene → Scene Time → output Seconds หรือ Frame ใส่เข้าสมการ ทุกอย่างเคลื่อนไหวอัตโนมัติ ตัวอย่าง

  • Wave animation: Position.Y = sin(Position.X + Time)
  • Rotation: rotate instances by Time × 30°
  • Growing animation: scale = clamp(Time − offset, 0, 1)
18

Fields & Attributes — ระบบ random ที่ควบคุมได้

— random กับ noise คือเพื่อนซี้ของ MG

ใช้ Random Value node พร้อม ID attribute ให้ instance แต่ละตัวมี seed ต่างกัน combinings Noise Texture 4D (3D + Time) ได้ animation organic แบบ studio

โปรเจคแนะนำ

ทำ "Particle text": ตัวอักษร 3D ที่กระจายเป็น cube 10000 ตัว แล้วประกอบกลับ — ใช้ GN ทั้งหมด ไม่มี keyframe ฝึก field ครบทุกแบบ

19

Simulation Zones — จำลองฟิสิกส์แบบ procedural

— ใหม่ใน Blender 4.x — เปลี่ยนเกมไปเลย

ตั้งแต่ 4.0 มี Simulation Input/Output ทำให้ GN จำเก็บค่าระหว่าง frame ได้ ใช้ทำ flocking, particle physics, growth simulation โดยไม่ต้องพึ่ง physics engine — ปรับได้ละเอียดกว่ามาก

● MODULE 06
06

Shading ที่เคลื่อนไหวได้

วัสดุไม่ใช่แค่สีและความมัน — ใน MG ยุคใหม่ shader คือตัวเล่าเรื่องอีกชั้น สร้าง procedural texture ที่ animation ได้

3 บท1.7hIntermediate+
20

Principled BSDF — เข้าใจทุก slider

— PBR shader ที่ใช้ได้ 90% ของงาน

เปิด Shader Editor (Shift+F3) เลือก object → ดู Principled BSDF default ค่า slider ที่สำคัญ

  • Base Color — สีจริงไม่มี light affect
  • Metallic — 0 = อโลหะ, 1 = โลหะ (ไม่มีระหว่างกลาง)
  • Roughness — 0 = กระจกเงา, 1 = ผิว matte
  • IOR — 1.45 (พลาสติก), 1.33 (น้ำ), 1.5 (แก้ว)
  • Emission — เปล่งแสงเอง (สำหรับงาน MG ใช้บ่อยมาก)
21

Procedural Textures — Noise, Voronoi, Wave, Magic

— texture ที่ไม่ต้องวาดเอง

5 texture ที่ใช้บ่อยที่สุดใน MG

  • Noise Texture — สำหรับ organic surface, cloud, terrain
  • Voronoi Texture — cell pattern, scale, F1/F2 distance
  • Wave Texture — ลายเส้น, ringed wood
  • Gradient Texture — สำหรับ ramp, mask
  • Musgrave — fBM noise สำหรับ terrain ละเอียด

ผูก output ผ่าน ColorRamp เปลี่ยน grayscale เป็นสีอะไรก็ได้

22

Animated shader — ใช้ Driver แทน keyframe

— ทำ shader ที่เปลี่ยนตามเวลา

Noise Texture มี input W (4D) ผูก driver ให้ W = #frame / 24 ทุกๆ วินาที texture จะ "เลื่อน" — ได้ effect liquid, energy, plasma ฟรีๆ

# Driver expression ที่ใช้บ่อย frame / 24 # เปลี่ยนทุก 1 วินาที sin(frame / 10) # คลื่นสั่น noise(frame * 0.1) # เปลี่ยนแบบสุ่ม smooth
● MODULE 07
07

Camera, Light, ภาษาภาพ

งานสวยเริ่มที่กล้องและแสง 90% ของ MG ที่ดูแพง = lighting ที่ดี — ไม่ใช่ render engine ที่ดี

4 บท2.0hIntermediate+
23

Camera Anatomy — focal length, sensor, DoF

— กล้องดิจิทัลที่เลียนแบบกล้องจริง

เพิ่มกล้อง (Add → Camera) Properties tab รูปกล้อง ค่าสำคัญ

  • Focal Length — 35mm = wide, 50mm = natural, 85mm+ = portrait/compress
  • Sensor Size — Full Frame (36mm) เป็น standard
  • Depth of Field → เปิด, ตั้ง Focus Object หรือ Distance
  • F-Stop — ค่าน้อย = เบลอเยอะ (1.4 ฉาก dramatic, 5.6 ทั่วไป)
24

Camera Animation — Follow Path + Track To

— กล้องที่เคลื่อนไหวสวยใน 3 ขั้นตอน
  • วาด Bezier Curve เป็นเส้นทางกล้อง
  • เลือกกล้อง → Add Constraint → Follow Path → target = curve
  • เพิ่ม Empty ที่ subject → กล้อง add Track To → target = empty
  • animate Path Animation ของ curve เพื่อ control speed
CINEMATIC TIP

เพิ่ม Camera Shake ด้วย Noise modifier บน F-Curve ของ camera location ค่าน้อยๆ (0.02) เพิ่มความรู้สึกจริงให้ทุก shot

25

3-Point Lighting — สูตรตาย

— Key, Fill, Rim — แค่ 3 ดวงเปลี่ยนทุกอย่าง
  • Key Light — ดวงหลัก (Area light, 200–500W) มุม 45° จากด้านหน้า
  • Fill Light — ลบเงาด้านตรงข้าม Key (1/3 ความสว่างของ Key)
  • Rim Light — ด้านหลัง subject แยกออกจาก background

ใช้ Area Light ใหญ่ๆ ดีที่สุดสำหรับ MG เงานุ่ม ไม่แข็งเหมือน Point Light

26

HDRI & World Lighting — environment ที่สมจริงทันที

— ฟรีและทรงพลัง

ดาวน์โหลด HDRI จาก polyhaven.com (ฟรี CC0) → Shader Editor → World → Environment Texture → load HDRI ใช้คู่กับ Background shader → strength 0.5–1.0

HDRI ให้ทั้ง ambient light และ reflection ในครั้งเดียว — เป็นวิธีเร็วที่สุดในการทำให้ฉากดู "อยู่ในที่ใดที่หนึ่งจริงๆ"

● MODULE 08
08

Render & Post — ส่งงานแบบ studio

ขั้นสุดท้ายคือเรื่องของรายละเอียด เลือก engine, ตั้งค่า output, จบด้วย compositor — pipeline ที่ใช้จริงในงาน commercial

4 บท2.0hPro
27

Eevee Next vs Cycles — เลือกอย่างไร

— ไม่มี engine ที่ดีที่สุด มีแต่ engine ที่เหมาะกับงาน
  • Eevee Next (Blender 4.2+) — real-time, ดีพอสำหรับ MG 80% รองรับ raytracing บางส่วน เร็วกว่า Cycles 10–50 เท่า
  • Cycles — path-tracing เต็มสูตร ใช้สำหรับ photoreal, glass, caustics, complex SSS
  • Workbench — สำหรับ preview/draft, เร็วสุด

กฎง่าย: ทำเสร็จใน Eevee ก่อน ถ้าไม่พอใจค่อยย้าย Cycles

28

Output Settings — ตั้งให้ถูกตั้งแต่ต้น

— save เป็น sequence ไม่ใช่ video
  • File Format → PNG หรือ OpenEXR (ไม่ใช่ MP4 โดยตรง)
  • Color Depth → 16 bit ขึ้นไปสำหรับ post
  • Compression → 15% (PNG)
  • หลัง render เสร็จ → ใช้ Video Sequence Editor หรือ FFmpeg compile เป็น MP4
เหตุผลที่ render เป็น sequence

ถ้า render ค้างกลางทาง คุณยังมีไฟล์ที่ render เสร็จแล้ว resume ต่อได้ ส่วนถ้า render เป็น MP4 ตรงๆ — ค้าง = เริ่มใหม่ทั้งเรื่อง

29

Compositor — เพิ่มชั้น polish ขั้นสุดท้าย

— Glare, Bloom, Chromatic Aberration, Grain

เปิด Compositing workspace → tick Use Nodes → effect ที่ใช้บ่อย

Render Layer → Glare (Fog Glow, threshold 1.0) ↓ Lens Distortion (dispersion 0.005) ↓ Color Balance (lift / gamma / gain) ↓ Film Grain (Add → Noise mix 0.05) ↓ Composite Output
30

Color Grading ด้วย AgX — workflow ปี 2026

— เลิกใช้ Filmic ได้แล้ว

Render Properties → Color Management → View Transform: AgX → Look: Medium High Contrast (เริ่มต้น) AgX จัดการ highlight roll-off, skin tone, saturation curve ดีกว่า Filmic ในทุกมิติ ไม่ต้อง LUT แล้ว

FINAL POLISH

ก่อน export ครั้งสุดท้าย: เพิ่ม vignette + grain เล็กน้อย + sharpen 0.5 ใน compositor — 3 อย่างนี้ทำให้ render ทุกตัวดู "filmic" ทันที

Motion Graphics ที่ดี ไม่ได้เกิดจาก plugin ราคาแพง
หรือ render engine ที่ดีที่สุด — มันเกิดจาก timing,
contrast และ การตัดสินใจที่ละเอียดทุกเฟรม
— ปรัชญาของหลักสูตร Motion/Craft

เคล็ดลับ มือโปร — สิ่งที่ต้องลอง

FROM THE FIELD
STUDIO PROVEN
TIP · 01

เริ่มที่ timing ก่อน look

render เป็น viewport flat shading ทำ animation ให้สมบูรณ์ก่อน อย่าเพิ่งเสียเวลากับ shader/light ตอนที่ animation ยังไม่ลงตัว — 70% ของเวลาควรอยู่ที่ Graph Editor

TIP · 02

Empty คือ เพื่อนซี้

ใช้ Empty เป็น parent, controller, focus point, target ของ track-to ทุกอย่าง animation ที่ดีคือลำดับชั้น (hierarchy) ที่ออกแบบดี ไม่ใช่ keyframe ที่ใส่เยอะ

TIP · 03

Render low-res ก่อน

50% resolution + 50% sample → ใช้เวลาแค่ 1/8 ของเดิม preview ทั้งเรื่องในเวลาเย็นๆ ก่อนสั่ง full render ตอนนอน เห็นปัญหาก่อนเสียเวลา

TIP · 04

Color palette 3 สี เท่านั้น

1 dominant + 1 secondary + 1 accent ห้ามเกิน MG ที่ดูแพง 99% มี palette จำกัด หาแรงบันดาลใจจาก Coolors.co หรือ palette ของหนังที่ชอบ

TIP · 05

Iteration ดีกว่า perfection

ทำให้เสร็จก่อน แล้วค่อยกลับมา polish ทำ v01 → v02 → v03 ดีกว่าทำ v01 perfect แล้วไม่กล้าเริ่ม v02 ใช้ Save Incremental (Ctrl+Alt+S) ทุกครั้งที่ผ่าน milestone

TIP · 06

ดูงานคนอื่นเยอะๆ

Motion Design Awards, The Rookies, Behance MG section, Greyscalegorilla studio reels — ดู 30 นาทีต่อวัน วิเคราะห์ว่าทำไมแต่ละช็อตทำงาน นี่คือ skill ที่ tutorial ไม่สอน
