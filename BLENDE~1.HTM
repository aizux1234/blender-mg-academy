โมดูล 01

ปฐมบท — เปิดโปรแกรมครั้งแรกอย่างถูกต้อง

หลายคนข้ามขั้นตอนนี้แล้วเสียเวลาทีหลังเป็นชั่วโมง การตั้งค่าก่อนเริ่มทำงานคล้ายๆ กับการลับมีดก่อนทำอาหาร — ใช้เวลา 15 นาทีก่อน ประหยัดเวลาภายหลังเป็นสิบเท่า

โมดูลนี้พูดถึงการติดตั้ง Blender, การจัดหน้าจอให้เหมาะกับงาน Motion Graphics, ปุ่มลัดที่ต้องคล่องตั้งแต่วันแรก และค่า scene ที่ตั้งให้ถูกตั้งแต่ต้น เมื่อจบโมดูลนี้คุณจะมี workspace ที่พร้อมทำงานจริง

01

ติดตั้ง Blender 4.x และตั้งค่าระบบครั้งแรก

— เลือกเวอร์ชันให้ถูก เปิดของที่จำเป็น

Blender ฟรี เปิดซอร์ส และอัปเดตเร็วมาก ใหม่ทุก 3 เดือน คำถามแรกของคนเริ่มต้นคือ "โหลดเวอร์ชันไหน?" — คำตอบคือ Long-term Support (LTS) ซึ่งคือเวอร์ชันที่ผ่านการเทสต์ และจะได้รับ patch แก้บั๊กไปอีก 2 ปี

ในขณะที่เขียนคู่มือนี้ Blender 4.2 LTS เป็นเวอร์ชันแนะนำที่สุด ดาวน์โหลดได้ที่ blender.org เลือก installer ตามระบบ — ติดตั้งเหมือนโปรแกรมทั่วไป ไม่มี trick

ตั้งค่าก่อนใช้งาน

เปิด Blender ครั้งแรกจะเห็น splash screen ปิดแล้วไป Edit → Preferences นี่คือสถานที่ที่เราจะอยู่กัน 5 นาทีต่อไปนี้

  1. System tab → Cycles Render Devices — ถ้าใช้ NVIDIA เลือก OptiX (ถ้าไม่มีให้เลือก CUDA), AMD เลือก HIP, Mac เลือก Metal — แล้ว tick การ์ดของคุณ ขั้นตอนนี้ทำให้ Cycles เร็วขึ้น 5–20 เท่า
  2. Save & Load — เปิด Auto Save และตั้ง Timer เป็น 2 นาที ปกติเป็น 10 นาที ซึ่งนานเกินไป
  3. Save & Load → Save Versions = 1 — เก็บไฟล์ .blend1 สำรองอัตโนมัติ ป้องกันงานหายเวลา save ทับโดยไม่ตั้งใจ
  4. Interface → Developer Extras — เปิด tick นี้ จะเปิดของหลายอย่างใน UI เช่น "Copy Data Path" สำหรับทำ driver ภายหลัง
  5. Input → Emulate 3 Button Mouse — เปิดถ้าใช้ trackpad หรือเมาส์ปุ่มเดียว (จำลองคลิ๊กกลางด้วย Alt+คลิ๊ก)
ทำไม

OptiX ใช้ RT Core ของ NVIDIA ในการคำนวณ ray-tracing โดยตรง เป็น hardware ที่ออกแบบมาเพื่อสิ่งนี้ การไม่เปิดใช้คือทิ้งความเร็วฟรีๆ

ลองทำดู

เปิด Preferences ตั้งค่าทั้ง 5 ข้อด้านบน ปิดหน้าต่าง — สังเกตว่า Blender จะ save preferences อัตโนมัติ ครั้งหน้าที่เปิดมาก็ใช้ได้เลย

ผิดบ่อย

มือใหม่หลายคนคิดว่า "ใช้ default ก็ได้ มีไรก็ค่อยแก้" — ผลคือ render ใน CPU เป็นชั่วโมงทั้งที่มี GPU แรงๆ อยู่ในเครื่อง ตั้งครั้งเดียวจบเลย

02

ทำความรู้จัก Interface — สนามทำงานของคุณ

— Blender ดูซับซ้อนเพราะรวมทุกอย่างในจอเดียว แต่จริงๆ ง่ายกว่าที่คิด

หน้าจอ Blender ประกอบด้วย Areas สี่เหลี่ยมที่แบ่งจอออกเป็นส่วนๆ แต่ละ Area เป็น Editor หนึ่งตัว ที่ทำหน้าที่เฉพาะ เช่น 3D Viewport, Outliner, Properties, Timeline

เคล็ดลับสำคัญ: คุณเปลี่ยน editor ใน area ใดก็ได้ โดยคลิ๊กไอคอนมุมซ้ายบนของ area นั้น และคุณ แบ่ง area โดยลากจากมุม หรือ รวม area โดยคลิ๊กขวาที่ขอบแล้วเลือก Join Areas

Editor หลัก 6 ตัวที่ต้องคล่อง

  • 3D Viewport — สนามทำงานหลัก ทำ modeling, animation, texturing ทุกอย่างเริ่มและจบที่นี่
  • Outliner — แสดง object ทั้งหมดในฉาก เป็น "สมุดรายชื่อ" ของ scene เปิด/ปิด/เลือกได้จากตรงนี้
  • Properties — แท็บไอคอนข้างล่างขวา ควบคุมทุกอย่างของ object/scene/render — ใช้บ่อยที่สุดรองจาก viewport
  • Timeline — แถบเล่น เห็น keyframe เป็นจุดสีเหลือง
  • Graph Editor — แสดง F-curve ของ animation — บทที่ 09 พูดเต็มๆ ตอนนี้แค่รู้ว่ามี
  • Shader Editor — สร้าง material ด้วย node — โมดูล 06

Workspaces — preset หน้าจอที่แตกต่าง

แถบบนสุดมีคำว่า Layout, Modeling, Sculpting, UV Editing, Texture Paint, Shading, Animation, Rendering, Compositing — แต่ละคำคือ Workspace ที่จัดหน้าจอให้เหมาะกับงานนั้นๆ

สำหรับ Motion Graphics, สามตัวที่ใช้บ่อย: Layout (ทำงานทั่วไป), Animation (มี Graph Editor + Dope Sheet), Shading (มี Shader Editor + viewport ใหญ่)

เคล็ดลับ

กด Ctrl + Space เพื่อ Maximize Area ของ editor ที่ cursor อยู่ — ขยายเต็มจอชั่วคราว กดอีกครั้งกลับมาขนาดเดิม ใช้บ่อยเวลาต้องการโฟกัส

ลองทำดู

ไปที่ workspace Animation สังเกตว่ามี Dope Sheet ด้านล่าง ลองคลิ๊กที่ไอคอนซ้ายสุดของ Dope Sheet เปลี่ยนเป็น Graph Editor — workspace นี้จะเป็น layout ที่ใช้ทำ MG ตลอด

03

Navigation — ปุ่มลัดที่ต้องคล่องในวันแรก

— ความเร็วในงานเริ่มที่นิ้วของคุณ ไม่ใช่ที่จอ

ผู้ใช้โปรแกรม design ส่วนใหญ่คลิ๊กเมนูเป็นหลัก แต่ Blender ออกแบบให้ใช้ คีย์บอร์ดเป็นหลัก ทุกคำสั่งสำคัญมีปุ่มลัด ฝึกให้คล่องภายในสัปดาห์แรก คุณจะเร็วขึ้น 3 เท่า

ที่สำคัญที่สุดคือ นิสัยการกดเมนูตามลำดับนิ้ว เช่น G แล้วตามด้วย X เพื่อ "เคลื่อนตามแกน X" — ไม่ใช่กดพร้อมกัน นี่ทำให้คำสั่งซ้อนกันได้

3 ปุ่ม transform ที่ใช้ทุกวัน

G Grab — เคลื่อนที่ R Rotate — หมุน S Scale

ลำดับปกติคือ: เลือก object → กด G → ขยับเมาส์ → คลิ๊กยืนยัน หรือกด Enter ถ้ากด X / Y / Z หลังจาก G จะจำกัดให้เคลื่อนที่แค่แกนนั้น เช่น G → Y = "เลื่อนแค่ Y"

ใส่ตัวเลขเลยก็ได้ เช่น G → X → 2 = "เลื่อนแกน X 2 เมตร" หรือ R → Z → 90 = "หมุนแกน Z 90 องศา" — เป็นวิธีปรับค่าที่แม่นยำที่สุด

Selection

ผิดบ่อย

คนที่มาจาก Maya/3ds Max มักจะกด Delete เพื่อลบ object — Blender ใช้ X หรือ Delete ก็ได้ แต่จะมี popup ถามยืนยัน ปกตินะ ไม่ใช่บั๊ก

ลองทำดู

เปิด Blender → เลือก Cube ที่อยู่ตรงกลาง → ฝึกชุดนี้ 5 รอบ: G → Z → 2 → Enter, R → X → 45 → Enter, S → 0.5 → Enter — เป้าหมายคือทำได้โดยไม่ดูคีย์บอร์ด

04

ตั้งค่าโปรเจคใหม่ — Resolution, FPS, Color

— ตั้งให้ถูกตั้งแต่ต้น ประหยัดเวลาทีหลัง

ก่อนสร้าง object ตัวแรก ตั้งค่า scene ก่อนเสมอ ขั้นตอนนี้ใช้เวลา 1 นาทีแต่ป้องกันปัญหาตอน render ซึ่งบ่อยครั้งทำให้ต้องแก้กันยาวๆ

Output Properties (ไอคอนเครื่องพิมพ์)

  1. Resolution — 1920×1080 (Full HD) สำหรับงานทั่วไป, 3840×2160 (4K) สำหรับ commercial — ตั้ง Resolution % = 100 ตอน final, ลดเป็น 50% ตอน preview
  2. Frame Rate — 24 fps สำหรับ cinematic, 30 fps สำหรับ social media (TikTok, IG), 60 fps สำหรับ smooth motion
  3. Frame Range — Start = 1, End = ความยาวที่ต้องการ × fps เช่น คลิป 5 วินาที 30fps = End 150

Output ของ render

  1. File Format — เลือก PNG หรือ OpenEXR สำหรับ animation — อย่าใช้ MP4 ตรงๆ
  2. Color → Color Depth — 16 หรือ 32 bit สำหรับ post-processing
  3. Output path — ตั้งให้เป็น folder ของโปรเจค ใส่ชื่อ prefix เช่น shot01_
ทำไมไม่ render เป็น MP4?

ถ้า Blender crash กลางทาง — render เป็น MP4 หาย ทั้งคลิป, render เป็น PNG sequence ยังมีไฟล์ที่เสร็จแล้ว resume ต่อได้ หลัง render เสร็จค่อยเอา PNG มาประกอบเป็น video ใน VSE หรือ FFmpeg

Color Management

ใน Render Properties → Color Management เปลี่ยน View Transform เป็น AgX

ที่ตั้งนี้สำคัญมากแต่หลายคนข้าม AgX คือ color science ใหม่ของ Blender 4.x ที่จัดการ highlight ดีกว่า Filmic เดิมในทุกมิติ ไฟ neon, แสงสะท้อน, skin tone — ทุกอย่างสวยขึ้นโดยไม่ต้องทำอะไรเพิ่ม

เคล็ดลับ

หลังตั้งค่าเสร็จ save เป็น startup file: File → Defaults → Save Startup File — Blender จะเปิดมาพร้อมค่าเหล่านี้ทุกครั้ง ไม่ต้องตั้งใหม่

โมดูลนี้คุณได้: ติดตั้ง Blender, เปิด GPU, จัดการ workspace, รู้จัก editor หลัก, ฝึกปุ่ม G/R/S, ตั้ง color management AgX และ save startup file

ถ้าทำตามครบ คุณมี environment ที่เหมือนกับมืออาชีพใช้กันแล้ว 80% — ที่เหลือคือทักษะที่จะเรียนในโมดูลถัดไป

โมดูล 02

หัวใจของ MG — Keyframe Animation

Motion Graphics ทุกชิ้นในโลก reduce ลงมาแล้วคือ "object กับ property ที่เปลี่ยนค่าตามเวลา" — และเครื่องมือที่บอก Blender ว่า "ค่าควรเปลี่ยนยังไง" คือ Keyframe

โมดูลนี้พูดเรื่อง keyframe ตั้งแต่แนวคิดพื้นฐาน จนถึงเทคนิคที่ studio ใช้จริงในงานทุกวัน เมื่อจบโมดูลนี้คุณจะเข้าใจว่า "การเล่าเรื่องผ่านการเคลื่อนไหว" ทำได้อย่างไร

05

Keyframe คืออะไร — แนวคิดก่อนปุ่มลัด

— ภาพ snapshot ของค่าหนึ่ง ณ เวลาหนึ่ง

คิดง่ายๆ: คุณบอก Blender ว่า "เฟรมที่ 1 ลูกบาศก์อยู่ที่ Z=0" แล้วบอกอีกครั้งว่า "เฟรมที่ 30 ลูกบาศก์อยู่ที่ Z=5" — Blender จะเติมเฟรม 2 ถึง 29 ให้เอง โดยคำนวณว่าระหว่างทางลูกบาศก์ควรอยู่ที่ไหน — กระบวนการนี้เรียกว่า interpolation

ดังนั้น keyframe ไม่ใช่ "การ์ดถ่ายรูป" ของ object ทั้งตัว — แต่เป็น snapshot ของ property หนึ่ง ณ เวลาหนึ่ง เช่น "Position.Z ที่ frame 30 = 5"

สิ่งสำคัญที่ต้องเข้าใจ: Blender แยก keyframe ของแต่ละ property คุณ key Location.X ไม่จำเป็นต้อง key Location.Y ด้วย คุณ key Rotation ไม่จำเป็นต้อง key Scale

วิธีใส่ keyframe

  1. เลื่อน playhead ไปยัง frame ที่ต้องการ (คลิ๊กที่ Timeline)
  2. ปรับ object ให้อยู่ที่ตำแหน่ง/ขนาด/มุมที่ต้องการ
  3. กด I (Insert)
  4. เลือก property ที่จะ key — ปกติเลือก Location, Rotation, Scale หรือ LocRotScale (ทั้ง 3 อย่าง)

หลังกดเสร็จ ใน Timeline จะมีจุดสีเหลืองปรากฏที่ frame นั้น คือ keyframe ที่เพิ่งใส่

I Insert keyframe Alt+I Delete keyframe คลิ๊กขวา property → Insert Keyframe
หมายเหตุ

คุณกด I ในที่ใดก็ได้ในหน้าจอ Blender — มันจะใส่ keyframe กับสิ่งที่ cursor อยู่ ถ้า cursor อยู่บน slider ของ Location.X มันจะใส่ key แค่ X

ลองทำดู

ใน scene เปล่า: ที่ frame 1 กด I → Location ที่ Cube — ขยับ playhead ไป frame 60 — กด G → Z → 5 → Enter — กด I → Location อีกครั้ง — กด Spacebar เพื่อ play — Cube ลอยขึ้น

06

Auto Keying — ดาบสองคม

— เปิดเมื่อต้องการ ปิดทันทีเมื่อจบ

ที่ Timeline header มีไอคอน วงกลมสีแดง ข้างปุ่ม play — นี่คือ Auto Keying เปิดแล้วจะ "auto-record" ทุกครั้งที่คุณขยับ object Blender จะใส่ keyframe ให้อัตโนมัติ ไม่ต้องกด I

ฟังดูดี — แต่อันตรายมาก ถ้าเปิดทิ้งไว้แล้วเผลอ scrub timeline หรือคลิ๊ก object โดยที่มันยังเลือกอยู่ keyframe จะแทรกเข้าไปในตำแหน่งที่ไม่ตั้งใจ ทำลาย animation ที่ทำมา

เมื่อไรควรเปิด Auto Keying

  • ตอนทำ animation แบบ character pose — pose ทีละ keyframe ต้องการเก็บทุกท่า
  • ตอนทำ blocking pass — วาง keyframe หลักๆ เร็วๆ

เมื่อไรควรปิด

  • ตลอดเวลา ที่ไม่ได้ทำ 2 อย่างข้างบน
  • เวลา ปรับ camera, ปรับ lighting — ไม่ต้องการให้กลายเป็น animation
  • เวลา เปิดงานเก่ามาดู — ขยับเล็กน้อยเพื่อตรวจ pose จะกลายเป็นพังงาน
เรื่องจริงเตือนใจ

มี studio หนึ่งใช้เวลา 2 ชั่วโมงตามหาว่า "ทำไม shot นี้ของ animation มันกระตุก" — คำตอบคือ animator คนหนึ่งเปิด Auto Keying ทิ้งไว้แล้วเผลอจับ object ระหว่างคุยโทรศัพท์ — keyframe ที่ frame 47 คลาดเคลื่อน 0.001 หน่วย

เคล็ดลับ

ตั้ง habit: ทุกครั้งที่จบ animation pose ให้ กดปิด Auto Keying ทันที เป็น muscle memory เดียวกับที่นักเขียนกด save

Keying Sets — เก็บคีย์เฉพาะที่อยากเก็บ

— ลด clutter ใน Graph Editor

ที่ Timeline header มี dropdown ชื่อ "Active Keying Set" เลือก preset ว่ากด I แล้วจะ key อะไรบ้าง

  • Location, Rotation, Scale — เก็บแค่อย่างเดียว เหมาะถ้ารู้แน่ๆ ว่า key อะไร
  • LocRotScale — เก็บทั้ง 3 (default ของ Blender) เก็บเยอะ ปลอดภัย
  • Available — เก็บเฉพาะ property ที่เคยมี keyframe อยู่แล้ว — มืออาชีพใช้ตัวนี้
  • Whole Character — สำหรับ character rig (โมดูลนี้ไม่ครอบคลุม)

ทำไม Available ถึงดีที่สุด

สมมุติคุณ animate แค่ Location.X และ Scale.Z ของลูกบาศก์ ถ้าใช้ LocRotScale Blender จะ key ทุก property ที่มีอยู่ — Graph Editor จะรกมาก เห็น curve นิ่งๆ ของ Y, Z, Rotation ที่ไม่จำเป็น

Available จะ key เฉพาะตัวที่เคยมี keyframe — Graph Editor สะอาด ดู curve ของ X กับ Z เท่านั้น เห็นชัดว่า animation อยู่ตรงไหน

workflow มาตรฐาน

ครั้งแรกที่ key property ใหม่ ใช้ คลิ๊กขวา → Insert Keyframe ที่ property นั้น (เพื่อกำหนดว่า property นี้จะ track) จากนั้นเปลี่ยน Keying Set เป็น Available แล้วใช้ I ตามปกติ

08

โปรเจคแรก — Bouncing Cube

— exercise ที่ครอบคลุมทุกอย่างใน 60 วินาที

โปรเจคนี้ทำในแล็บแรกของทุกคอร์ส animation มา 30 ปีแล้ว — เพราะมัน เรียบง่ายแต่ซ่อนทุกหลักการสำคัญ ทำตามจริงๆ อย่าข้าม

เป้าหมาย

Cube ตกจากความสูง 5 หน่วย กระเด้งขึ้น 3 ครั้งโดยแต่ละครั้งสูงน้อยลงเรื่อยๆ จนหยุดที่พื้น เวลาทั้งหมด 60 frames (2.5 วินาทีที่ 24fps)

ขั้นตอน

  1. เริ่มจาก scene เปล่า — ลบ Cube default แล้ว Add → Mesh → Cube ใหม่ที่ตำแหน่ง origin
  2. กด N เพื่อเปิด Sidebar ใน viewport — เห็น Location, Rotation, Scale
  3. เลื่อน playhead ไป frame 1 — ตั้ง Location Z = 5 — กด I → Location
  4. frame 20 — ตั้ง Z = 0 — กด I (เด้งครั้งแรก)
  5. frame 35 — ตั้ง Z = 2.5 — กด I (กระเด้งกลับขึ้น)
  6. frame 45 — ตั้ง Z = 0 — กด I (เด้งครั้งที่ 2)
  7. frame 53 — ตั้ง Z = 1 — กด I (กระเด้งครั้งที่ 3)
  8. frame 60 — ตั้ง Z = 0 — กด I (จบ)
  9. กด Spacebar เล่น — ดูว่า cube กระเด้งแบบ "ลื่นๆ" เพราะ default interpolation คือ Bezier

เพิ่ม squash & stretch (ขั้นต่อไป)

ตอนที่ cube กระทบพื้น (frame 20, 45) ตั้ง Scale Z = 0.6, X = 1.4, Y = 1.4 แล้ว key Scale — ที่ frame 1 กลับมา Scale 1,1,1 — เด้งไปกี่ครั้งก็ตั้ง squash ตอนกระทบทุกครั้ง

ทำไมต้อง squash

วัตถุที่ "นุ่ม" (เช่นลูกยาง) จะบีบเมื่อกระทบ การใส่ squash ทำให้ดูมีน้ำหนักและ bounce แบบมีชีวิต — ลองทำทั้งสองแบบเปรียบเทียบจะเห็นความต่างทันที

ลองทำดู

หลังทำเวอร์ชันแรกเสร็จ save as เป็นไฟล์ใหม่ bouncing_v02.blend แล้วลอง: เปลี่ยน timing ให้กระเด้งแน่นขึ้น (15, 25, 32, 38), เพิ่ม rotation Y ที่หมุนตอนกระเด้ง, เปลี่ยน scale squash ให้แรงขึ้น เล่นกับ values สนุกๆ

โมดูลนี้คุณได้: เข้าใจว่า keyframe คือ snapshot ของ property, รู้วิธี insert ด้วย I, รู้ว่าเมื่อไรควรเปิด/ปิด Auto Keying, ใช้ Keying Set แบบ Available, และทำ animation จริงเสร็จ 1 ชิ้น

นี่คือทักษะ 80% ของ animator — ที่เหลือ 20% คือเรื่อง timing และ curve ที่อยู่ในโมดูลถัดไป

โมดูล 03

Graph Editor & ศิลปะของ Easing

นี่คือโมดูลที่แยกมือใหม่กับมือโปร — ความแตกต่างระหว่าง animation ที่ดู "งั้นๆ" กับ animation ที่ดู "แพง" 95% อยู่ที่ curve ของ keyframe ไม่ใช่ที่ตำแหน่ง keyframe

Graph Editor คือเครื่องมือที่ให้คุณควบคุม curve เหล่านั้น เมื่อจบโมดูลนี้คุณจะเลือก easing ได้อย่างมั่นใจ และเข้าใจว่าทำไม "ความเร็วคงที่" ถึงดูเสมือนวิดีโอเกมยุค 80

09

F-Curves — ภาษาของ animation

— ทุก animation คือกราฟ value(time)

เปิด Graph Editor (workspace Animation มีอยู่แล้ว) คุณจะเห็นเส้นโค้งหนึ่งหรือหลายเส้น แสดง property ตามแกนเวลา

แกน X = frame number (เวลา) — แกน Y = value ของ property เช่นถ้าเป็น Location.Z, Y = ความสูง

แต่ละ keyframe คือ "จุด" บน curve มี handle สองข้างยื่นออกมา (เหมือน Bezier curve ใน Figma) handle นี่แหละที่กำหนดว่า curve ระหว่าง keyframe จะ "นุ่ม" หรือ "ตึง" ยังไง

F-Curve เปรียบเทียบ — linear (เส้นประ) vs Bezier (เส้นเต็ม)

ปุ่มลัดที่ต้องคล่องใน Graph Editor

T Interpolation Type (Bezier/Linear/Constant) V Handle Type (Auto/Vector/Free) G เคลื่อน keyframe S scale (ยืด/บีบ timing)

การเลือก T → Bezier → Easing → ... เป็นที่ที่เกิด "magic" ของ MG ดูบทถัดไปสำหรับเมนูทั้งหมด

ลองทำดู

เปิดไฟล์ Bouncing Cube จากบทก่อน → เปิด Graph Editor → กด A เลือก keyframe ทั้งหมด → กด T → Linear → เล่น — สังเกตว่า cube เคลื่อนเป็นเส้นตรง ดูเหมือนหุ่นยนต์ — กลับเป็น Bezier ดู ความแตกต่างคือ "ความมีชีวิต"

10

Easing — เส้นแบ่งระหว่างมือใหม่กับมือโปร

— ของจริงไม่เคลื่อนที่เร็วคงที่

ลองสังเกตของจริง: รถออกตัว — เริ่มช้าๆ ค่อยๆ เร่งขึ้น รถเบรค — เร็วก่อน ค่อยๆ ช้าลง ลูกบอลตก — เริ่มช้า ตกเร็วขึ้นเรื่อยๆ ตามแรงโน้มถ่วง

ในชีวิตจริงแทบไม่มีอะไรเคลื่อนที่ด้วย ความเร็วคงที่ Linear interpolation ทำให้ animation ดูเสมือนเครื่องจักรในวิดีโอเกมยุค 80

เลือก Easing ใน Blender

เลือก keyframe ใน Graph Editor → กด T → เห็นเมนูแบ่งเป็น 3 หมวด:

  • Constant — ไม่เปลี่ยนค่าจน frame ถัดไป (สำหรับ animation แบบ stop-motion)
  • Linear — เร็วคงที่ (ใช้น้อย)
  • Bezier — smooth curve (default)

ภายใน Bezier มีตัวเลือกย่อย Easing ที่ละเอียดกว่า — กด T → Easing (by strength):

3 easing หลัก — ใช้แตกต่างกันตามบริบท

เลือก strength ของ easing

ในเมนู easing คุณจะเห็นชื่ออย่าง Sinusoidal, Quadratic, Cubic, Quartic, Quintic, Exponential, Circular นี่คือระดับ "ความรุนแรง" ของ curve — Sine เบาสุด, Exponential แรงสุด

  • Sinusoidal / Quadratic — เนียน เบา เหมาะกับ camera move, gentle UI
  • Cubic / Quartic — เริ่มมี "punch" — ใช้ทั่วไปสำหรับ MG
  • Quintic / Exponential — แรง snap — เหมาะกับ "พุ่ง", "เด้ง" แบบมีพลัง

กฎสำหรับ MG: Quintic Out เป็นค่าเริ่มต้นที่ดีมากสำหรับ 80% ของ animation

Easing พิเศษ — Back, Elastic, Bounce

  • Back — overshoot ก่อนถึงจุดหมาย (กระดิกเล็กน้อย) — ใช้กับ logo reveal, button bounce
  • Elastic — สั่นแกว่งหลายรอบ — ใช้กับ cartoon พลาด สะดุดตามาก
  • Bounce — กระเด้งหลายรอบ — cliché ที่ใช้บ่อยเกินไปในงาน MG ปี 2010 ระวัง
กฎทอง

เร็วในตอนต้น ช้าในตอนจบ — Ease Out (Quint หรือ Cubic) เป็น default ที่ปลอดภัยที่สุด ใช้ได้กับ camera, position, scale, opacity ทุกอย่างใน UI

12 หลัก Animation จาก Disney — ใช้ใน MG ได้จริง

— จาก The Illusion of Life (1981) ถึง MG ปี 2026

หลักของ Disney เขียนสำหรับ character animation แต่ ใช้ได้กับ motion ทุกแบบ — รวมถึง MG ที่ไม่มีตัวการ์ตูน คือเหตุผลที่ studio ระดับโลกยังศึกษา 40 ปีหลังตีพิมพ์

4 หลักที่สำคัญที่สุดสำหรับ MG:

1. Squash & Stretch — บีบและยืด

วัตถุที่ "อ่อน" จะเปลี่ยนรูปร่างเมื่อมี force การใส่ S&S ทำให้รู้สึกถึงน้ำหนักและความเร็ว ลูกบอลกระเด้ง: บีบตอนกระทบ, ยืดยาวตอนพุ่งเร็วๆ

ใน Blender: animate Scale ที่ swap แกน — ตอนพุ่งขึ้น Scale.Z เพิ่ม Scale.X/Y ลด, ตอนกระทบ Scale.Z ลด Scale.X/Y เพิ่ม รักษา volume ให้คงที่

2. Anticipation — ขยับสวนทางก่อน

ก่อนตัวการ์ตูนกระโดด มันต้อง "ย่อ" ก่อน ก่อนนักธนูยิงต้องดึงคันธนู — นี่ไม่ใช่ความจำเป็นทางฟิสิกส์ แต่ทำให้คนดูคาดเดาได้ว่ากำลังจะเกิดอะไร

ใน MG: ก่อน object พุ่งเข้า ขยับมัน ออกเล็กน้อย 5–10 frames ก่อน — เพิ่มความรู้สึก "พลัง" มหาศาล

3. Follow Through & Overlap — ปลายตามทีหลัง

ส่วนที่หนักกว่าหยุดก่อน ส่วนที่เบากว่าตามมา หางม้า ผม ผ้า เสื้อโค้ท — animate ให้ ปลายตามมาช้ากว่าตัวหลัก 3–5 frames

4. Slow In, Slow Out — easing

คือสิ่งที่เราเรียนในบท 10 — Disney เรียกชื่อนี้ก่อนคำว่า easing จะแพร่หลาย หลักการเดียวกัน

ทำไมหลัก 4 ข้อนี้ถึงสำคัญที่สุด

เพราะมัน ทำให้ object ที่ไม่มีชีวิตดูเหมือนมีชีวิต — และ MG ที่ดีคือการทำให้กราฟิกธรรมดาดูมีพลัง ไม่ใช่ทำให้ตัวการ์ตูนเดิน

ลองทำดู

เปิด Bouncing Cube ใส่ทั้ง 4 หลัก: squash ตอนกระทบ, anticipation ก่อนเด้งครั้งแรก (ย่อตัว 5 frames), follow through (ใส่ Empty เล็กๆ ตามหลัง cube ช้ากว่า 3 frames), Ease Out ทุก curve — เปรียบเทียบกับเวอร์ชันแรก

โมดูลนี้คุณได้: เข้าใจ F-Curve เป็นภาษาของ animation, เลือก easing ตามบริบท, รู้ว่า Linear มักจะผิด, เริ่มใช้ Quintic Out เป็น default, และนำ 4 หลักของ Disney มาใช้ในงาน abstract ได้

หลังจากนี้คุณ "ดู" animation ของคนอื่นแล้วจะเห็นว่ามันใช้ curve อะไร นั่นคือสายตาของ animator — ทักษะที่ developed มาจากการสังเกต

โมดูล 04

Drivers & Constraints — automation

ถึงจุดหนึ่ง keyframe จะไม่พอ คุณจะต้องการ "ผูก property หนึ่งให้ขึ้นกับอีก property" — เช่น "ล้อรถหมุนตามระยะทางที่รถเคลื่อน" — Drivers และ Constraints คือเครื่องมือสำหรับสิ่งนี้

เมื่อจบโมดูลนี้คุณจะลด "งาน manual" ได้มหาศาล และเริ่มสร้าง controller แบบที่ studio ใช้ในงานจริง

12

Constraints — ลูกศรที่บอก "ตามฉันไป"

— ผูก object ตัวหนึ่งให้ขึ้นกับอีกตัว

เลือก object → ที่ Properties panel ด้านขวา หาแท็บรูป โซ่ (Object Constraints) — กด Add Object Constraint เลือกชนิด

Constraint สำคัญสำหรับ MG

  • Copy Location — ตามตำแหน่ง target 100% หรือบางแกน — ทำ "เงา" ตามตัว
  • Copy Rotation — หมุนตาม target — ใช้ทำ gimbal, dial
  • Copy Scale — ขนาดตาม target
  • Track To — หันหน้าหา target ตลอด — ใช้กับ camera, spotlight, eye-tracking
  • Follow Path — เดินตาม curve ที่วาดไว้ — ทำ camera fly-through ได้สวย
  • Limit Location/Rotation/Scale — กำหนดขอบเขตการเคลื่อนที่ — กัน object หลุดเฟรม
  • Child Of — parent แบบมี keyframe ได้ (เปลี่ยน parent กลางทาง — เช่น object หยิบไม้ขึ้นแล้ววาง)
หมายเหตุสำคัญ

Constraint ทำงาน ทับ ค่าจริงของ object — ถ้าใส่ Copy Location เต็ม object จะติด target ขยับด้วยตัวเองไม่ได้แล้ว ใช้ Influence slider ลดสัดส่วน หรือ animate Influence (key) เพื่อ "เปิด-ปิด" constraint ตามเวลา

ลองทำดู

สร้าง Camera + Empty (Add → Empty → Plain Axes) → เลือก Camera → Add Constraint → Track To → Target = Empty → Track Axis = -Z, Up = Y → ขยับ Empty เล็กๆ — Camera หันตาม นี่คือพื้นฐานของ camera rig ที่ใช้ในงานจริง

13

Drivers — คณิตศาสตร์ขับเคลื่อน animation

— เปลี่ยน property หนึ่งให้ขึ้นกับสมการ

Constraint จัดการแค่ Loc/Rot/Scale ของ object Drivers ทำได้กับ property อะไรก็ได้ — สีของ material, intensity ของ light, slider ใน modifier, ตัวเลขใน geometry node

วิธีใส่ driver:

  1. คลิ๊กขวาที่ property ใดๆ ที่อยากให้ animate — เช่น slider ของ Light Power
  2. เลือก Add Driver
  3. หน้าต่าง driver editor เปิดขึ้น
  4. กำหนด Variable — ค่าที่ใช้คำนวณ ปกติคือ Transform Channel ของ object อื่น
  5. เขียน Expression — สมการที่ใช้ค่า variable คำนวณค่าใหม่

Driver pattern ที่ใช้บ่อย

# Pattern 1: ล้อหมุนตามตำแหน่งของรถ Driver: Wheel.rotation_euler[1] Variable: var = Car.location.x Expression: -var / 0.5 # 0.5 = รัศมีล้อ # Pattern 2: scale ตามระยะห่างจาก empty Driver: Object.scale (ทุกแกน) Variable: var = distance(Object, Empty) Expression: 1 + var * 0.1 # Pattern 3: light strength ตาม custom property Driver: Light.energy Variable: var = Empty["intensity"] # custom property Expression: var * 1000
ทำไม Drivers สำคัญ

คุณ "ผูกระบบ" ครั้งเดียว แล้วควบคุมทุกอย่างผ่าน controller ตัวเดียว — เปลี่ยนค่า Empty ตัวเดียว มี object 50 ตัวเปลี่ยนตาม นี่คือพื้นฐานของ rigging และ procedural workflow

Empty + Custom Property = Controller มืออาชีพ

— rig MG ใน 5 นาที

เทคนิคที่ studio ใช้กันทั่วไป: ใช้ Empty เป็น "หน้าจอควบคุม" — ใส่ custom property หลายตัว แล้ว driver ทุก object ในฉากให้ขึ้นกับ property เหล่านั้น

วิธีใส่ Custom Property

  1. เลือก Empty
  2. ที่ Properties panel แท็บ Object → เลื่อนลงสุด → Custom Properties
  3. กด New — สร้าง property ใหม่
  4. คลิ๊กไอคอน gear → Edit — ตั้งชื่อ (เช่น "intensity"), Min, Max, Default

วิธี driver ให้ใช้ค่านี้

  1. ที่ property ใดๆ ที่ต้องการ → คลิ๊กขวา → Add Driver
  2. Variable Type = Single Property
  3. Object = Empty, Path = ["intensity"]
  4. Expression = var (หรือคูณด้วยอะไรก็ได้)
เคล็ดลับมือโปร

สร้าง Empty หนึ่งตัวชื่อ "CTRL" วางไว้กลาง scene ใส่ custom property 5–10 ตัว: logo_reveal (0–1), camera_shake (0–1), bg_brightness (0–1) ฯลฯ — driver ทุกอย่างในฉากผ่าน slider เหล่านี้ ตอน present ลูกค้าก็ลาก slider ตัวเดียวเห็นทั้งซีนเปลี่ยน

ลองทำดู

สร้าง Empty + custom property "wave" (0–1) → ใส่ cube 10 ตัวเรียง → driver ให้ Z ของ cube แต่ละตัว = sin(wave * 6.28 + index * 0.5) โดย index เปลี่ยนตามตำแหน่งของ cube — animate "wave" จาก 0 ไป 1 ใน 60 frames — ได้คลื่นเป็นระลอกของ cube 10 ตัวด้วย slider เดียว

โมดูลนี้คุณได้: ใช้ Constraint ผูก object เข้าด้วยกัน, สร้าง Driver ด้วยสมการ, ใช้ Empty + Custom Property เป็น controller — นี่คือพื้นฐานของ rigging ทั้งหมด

เมื่อชินกับเทคนิคพวกนี้คุณจะรู้สึกเหมือน "เขียน programming" — เพราะมันคือการคิดเชิงระบบ ไม่ใช่การ keyframe ทีละอัน

Geometry Nodes — procedural MG

นี่คือเครื่องมือที่เปลี่ยนวงการ MG ในยุค 2020s ก่อนมี GN ถ้าต้องการให้ object 1000 ตัวกระจายเป็นรูปทรงเฉพาะ ต้อง keyframe หรือเขียน Python — ใช้เวลาเป็นชั่วโมงและแก้ไขยาก

GN ให้คุณ "เขียนสูตร" ผ่าน node graph — เปลี่ยน 1000 เป็น 10000 ในวินาทีเดียว เปลี่ยนรูปทรงเหมือนเปลี่ยนสไลด์ใน Figma โมดูลนี้ใหญ่กว่าโมดูลอื่น เพราะ GN มี depth มาก แต่อย่างน้อยอ่านครบจะเปิดประตูสู่งาน MG ระดับโลก

Geometry Nodes คืออะไร — ทำไมเปลี่ยนเกม

— programming แต่ใช้ node ไม่ใช้ code

เลือก object → ที่ Properties → แท็บ Modifiers (รูปประแจ) → Add Modifier → Geometry Nodes — หรือเปิด workspace Geometry Nodes ที่มี editor พร้อม

Editor นี้คือ node graph ที่บอก Blender ว่า "เปลี่ยน geometry ของ object นี้ตามสูตรนี้" — input geometry → modify → output geometry

ทำไม procedural ถึงดีกว่า manual

  • แก้ไขได้ตลอด — เปลี่ยน input ค่าหนึ่ง ทุกอย่างที่ต่อกันคำนวณใหม่
  • scalable — ใส่ object 100 หรือ 100,000 ใช้ resource ต่างกันไม่กี่เท่า
  • reusable — บันทึกเป็น "node group" เอาไปใช้กับ object อื่น
  • ไม่ทำลายของเดิม — เป็น modifier ปิดได้ ลบได้ ของเดิมยังอยู่
เปรียบเทียบให้เห็นภาพ

การใช้ keyframe คือ วาดทุกเฟรม — การใช้ GN คือ เขียนสูตรให้เครื่องวาดให้ — แก้สูตรได้, run กี่เฟรมก็ได้, แก้แล้วทุกเฟรมเปลี่ยนพร้อมกัน

Distribute & Instance — pattern พื้นฐานที่ใช้ตลอด

— สร้างป่าจาก mesh ตัวเดียว

นี่คือ flow ที่เห็นใน 90% ของงาน GN — เรียนรู้ครั้งเดียวใช้ตลอด

Setup

  1. สร้าง Plane (Add → Mesh → Plane) — นี่คือ "พื้นผิว" ที่จะกระจาย instance บน
  2. เพิ่ม Geometry Nodes modifier — กด New สร้าง node group
  3. สร้างอีก object เป็น "instance" เช่น Add → Mesh → Cone (ทำเป็นต้นไม้)
  4. ที่ Cone ตั้ง Viewport Display → แสดงผลแบบ Wireframe หรือ Hide เพราะเราใช้แค่ shape

Node graph

Group Input (geometry: plane) ↓ Distribute Points on Faces density: 50 ↓ Instance on Points ↑ instance: Object Info → Cone ↓ Rotate Instances rotation: Random Value (0 → 360°, axis Z) ↓ Scale Instances scale: Random Value (0.7 → 1.3) ↓ Group Output

วาง node ตามลำดับ → connect socket — ผลลัพธ์: ป่า 50 ต้นกระจายแบบ random หมุนคนละทิศ ขนาดต่างกันเล็กน้อย

การเชื่อม node

ลากจาก output socket (วงกลมขวา) ไปที่ input socket (วงกลมซ้าย) — สีของ socket บอก data type: เขียว = geometry, ม่วง = vector, เทา = number, สีฟ้า = boolean

เคล็ดลับ

กด Ctrl + เลื่อนเมาส์ บน socket ของ Distribute Points → "Density" — slider ใน Group Input จะปรากฏ ทำให้คุณ animate density ได้ — animate จาก 0 → 100 ป่าค่อยๆ "เติบโต"

Simulation Zones — physics แบบ procedural

— ใหม่ใน 4.0+ — เปลี่ยนเกมไปเลย

ก่อน 4.0, GN ไม่ "จำ" ค่าระหว่าง frame ทุก frame คำนวณใหม่ตั้งแต่ต้น — Simulation Zones แก้ปัญหานี้

Add → Simulation → Simulation Zone — สร้าง 2 node: Simulation Input และ Simulation Output — ทุกอย่างระหว่างนี่คือ "loop" ที่ทำงานทุก frame โดยใช้ output จาก frame ก่อน

  • Particle physics — gravity, velocity, collision
  • Flocking / boids — ฝูงนก ฝูงปลา ที่หลีกกันเอง
  • Growth simulation — กิ่งไม้ที่งอกตามเวลา, vine ที่เลื้อย
  • Cellular automata — Conway's Game of Life
หมายเหตุ

Simulation Zone ทำให้ scene หนักขึ้น เพราะต้อง "play through" จาก frame 1 เพื่อคำนวณ — ใช้ Bake (gear icon → Bake) เพื่อเก็บผลลัพธ์ ทำให้ scrub timeline ได้เร็ว

โมดูลนี้คุณได้: เข้าใจ paradigm procedural, สร้าง Distribute + Instance flow, ใช้ Scene Time เป็น input, ผูก Random กับ Noise 4D, และรู้จัก Simulation Zones

Shading — วัสดุที่เคลื่อนไหวได้

วัสดุไม่ใช่แค่สีและความมัน ใน MG สมัยใหม่ shader คือ ตัวเล่าเรื่องอีกชั้น สร้าง texture ที่ animation ได้เอง สร้างผิวที่ "หายใจ" — ทำให้กราฟิกที่เรียบง่ายดูพิเศษ

โมดูลนี้สั้นแต่สำคัญ — เรียน Principled BSDF ที่ครอบคลุม PBR ทั้งหมด, procedural texture 5 ตัวหลัก, และเทคนิค animate shader ด้วย driver

20

Principled BSDF — เข้าใจทุก slider

— PBR shader ที่ใช้ได้ 90% ของงาน

เปิด workspace Shading → เลือก object → ที่ Shader Editor มี Principled BSDF ต่อกับ Material Output อยู่แล้ว

BSDF = Bidirectional Scattering Distribution Function — แค่คำเรียก math ที่อธิบายว่าแสงสะท้อนยังไง คุณไม่ต้องเข้าใจ math แต่ต้องเข้าใจ slider ที่สำคัญ

Slider หลัก

  • Base Color — สีจริงของวัตถุ (albedo) เลือก color picker หรือ texture
  • Metallic — 0 = อโลหะ (พลาสติก, ผิว, ไม้), 1 = โลหะ (ทอง, เงิน, เหล็ก) — ไม่มีระหว่างกลาง ในโลกจริง
  • Roughness — 0 = กระจกเงา, 1 = ผิว matte ทั้งหมด — slider สำคัญที่สุดในการกำหนด "feel" ของวัสดุ
  • IOR — Index of Refraction — 1.45 พลาสติก, 1.33 น้ำ, 1.5 แก้วทั่วไป, 2.4 เพชร
  • Transmission — 0 = ทึบ, 1 = โปร่งใส (กระจก, น้ำ)
  • Emission — แสงเปล่งออกเอง — ใช้บ่อยใน MG สำหรับ neon, screen, glow
  • Subsurface — แสงทะลุผ่านผิว (skin, ขี้ผึ้ง, นม)

กฎ PBR ที่ต้องจำ

  1. Metallic = 0 หรือ 1 เท่านั้น — ค่าระหว่างกลางใช้แค่ตอน "ผิวขรุขระ" เช่นโลหะที่มีฝุ่นเกาะ
  2. Roughness ≠ 0 เพราะของจริงไม่มีอะไรเรียบสมบูรณ์ — ตั้ง 0.05–0.1 อย่างน้อย
  3. Base Color ของโลหะ = สีของแสงสะท้อน — ทอง = #f4d03f, เงิน = #c0c0c0
ลองทำดู

สร้างทรงกลม 4 ตัวเรียงกัน — ตั้ง Metallic เป็น 0 ทั้งหมด ส่วน Roughness ตั้งเป็น 0, 0.3, 0.6, 1.0 — render — เห็นเฉดที่ต่างกัน นี่คือ "ภาษา" ของวัสดุ ฝึกตามาให้แม่น

Procedural Textures — texture ที่ไม่ต้องวาดเอง

— สร้างด้วย math ไม่ใช้รูป

Add → Texture ใน Shader Editor — มี texture หลายแบบที่สร้างจาก math ทั้งหมด ไม่ต้อง UV, ไม่ต้องเปลือง memory

5 texture ที่ใช้บ่อยที่สุด

  • Noise Texture — ลายสุ่ม smooth — ใช้ทำผิว organic, terrain, cloud
  • Voronoi Texture — cell pattern (เซลล์เปลือกหิน) — มี mode F1/F2 สำหรับลายต่างกัน
  • Wave Texture — เส้นคลื่น — ทำลายไม้, ringed pattern
  • Gradient Texture — gradient ตามแกน — ใช้กับ ColorRamp ทำ ramp
  • Magic Texture — pattern abstract สวยๆ — ใช้กับ MG psychedelic

การใช้งาน — ผ่าน ColorRamp

output ของ texture เหล่านี้คือ ค่าตัวเลข 0–1 ผูกผ่าน ColorRamp เพื่อแปลงเป็นสีตามต้องการ

เคล็ดลับ

ใช้ Voronoi mode Smooth F1 ผสมกับ ColorRamp ที่มีจุด stop หลายจุด ได้ผิวที่ดูเหมือน "หิน organic" ใน 30 วินาที

Animated Shader — texture ที่เคลื่อนไหว

— driver บน W ของ texture

Noise และ Voronoi มี input ที่ชื่อ W (4D coordinate) ผูก driver: frame / 24 — ทุกวินาทีเปลี่ยน 1 หน่วย texture จะ "เลื่อน" ตามเวลา

ผลลัพธ์: shader ที่ดู liquid, energy, plasma, electricity — ฟรีโดยไม่ต้องทำ animation อะไรเพิ่ม

# Driver expression สำหรับ shader animation frame / 24 # เปลี่ยนทุก 1 วินาที (ปกติ) frame / 12 # เร็วขึ้น 2 เท่า sin(frame / 10) # คลื่นไป-กลับ noise(frame * 0.1) # เปลี่ยนแบบสุ่ม smooth
ลองทำดู

สร้างทรงกลม → Material → Noise Texture → ColorRamp ไป Emission ของ Principled BSDF (ค่า 5) → Driver ที่ W ของ Noise: frame / 30 → render animation 60 frames — ได้ลูกบอลที่เปล่งแสงด้วย texture เคลื่อนไหว สวยมาก

โมดูลนี้คุณได้: ใช้ Principled BSDF ทุก slider, ใช้ procedural texture 5 ตัว, ผสม ColorRamp, และทำ animated shader ด้วย driver

Camera Anatomy — focal length, sensor, DoF

— กล้องดิจิทัลเลียนแบบกล้องจริง

Add → Camera — มันคือ object ที่อ้างอิงไปสู่จุดมอง render

เลือก camera → Properties → แท็บรูปกล้อง — ค่าสำคัญ:

Lens / Focal Length

  • 18–24mm — wide angle, distort สูง — ใช้ทำ landscape, dramatic perspective
  • 35mm — wide แต่ natural — เหมาะกับ scene กว้างๆ
  • 50mm — natural สุด — เหมือนตามองตรง
  • 85mm — portrait — compress space, blur background
  • 135mm+ — telephoto — flatten perspective, ใช้ทำ "cinematic compression"

ทดลอง: scene เดียวกัน เปลี่ยน focal length 24mm → 50mm → 100mm — ขนาด object เท่าเดิมแต่ depth ของฉากเปลี่ยน นี่คือ "ภาษา" ของ camera

Depth of Field

  1. เปิด Depth of Field tick
  2. ตั้ง Focus Object = object ที่ต้องการให้คม (หรือใช้ Distance ตัวเลขก็ได้)
  3. ตั้ง F-Stop — เลขน้อย = blur มาก, เลขเยอะ = คมหมด
  • F/1.4 — blur dramatic — portrait, beauty shot
  • F/2.8 — blur ปานกลาง — most cinematic
  • F/5.6 — บางส่วน blur — interview, product
  • F/16+ — คมเกือบหมด — landscape
หมายเหตุ

F-Stop ใน Blender เลียนแบบกล้องจริง — แต่ถ้าตั้ง Focus Object เป็น Empty แล้ว animate Empty คุณจะได้ rack focus (focus เลื่อน) แบบหนัง

Camera Animation — Follow Path + Track To

— camera fly-through ใน 3 ขั้นตอน

เทคนิค "camera ตามเส้นทาง พลางมองที่จุดเดียว" ใช้ใน 80% ของ camera move ระดับมืออาชีพ

ขั้นตอน

  1. วาด Bezier Curve เป็นเส้นทาง — Add → Curve → Bezier
  2. แก้ shape ของ curve โดยกด Tab เข้า Edit Mode → ขยับ control point
  3. เลือก camera → Add Constraint → Follow Path → Target = curve
  4. เพิ่ม Empty (Add → Empty) ที่ subject — นี่คือจุดที่ camera มอง
  5. camera → Add Constraint → Track To → Target = Empty → Track Axis = -Z, Up = Y
  6. ที่ curve Properties → Path Animation tab — animate Evaluation Time เพื่อ control speed

ตอนนี้ camera จะเดินตาม curve โดยมองที่ Empty ตลอด animate Evaluation Time จาก 0 → 100 ใน 120 frames — ได้ camera move ที่ professional

เคล็ดลับ cinematic

เพิ่ม Camera Shake: ที่ camera location ใส่ Noise modifier บน F-Curve (Graph Editor → เลือก curve → กด N → Modifiers tab → Add → Noise) — strength 0.02, scale 5 — เพิ่มความรู้สึก hand-held ทันที

ลองทำดู

วาง Cube ตรงกลาง → ตั้ง camera ขึ้น Follow Path + Track To ตามขั้นตอน → animate Evaluation Time 0→100 ใน 120 frames → render — เห็น camera fly-around Cube มองที่ subject ตลอด

3-Point Lighting — สูตรตายของวงการภาพ

— Key, Fill, Rim — แค่ 3 ดวงเปลี่ยนทุกอย่าง

สูตรนี้ใช้มา 100 ปีในวงการภาพยนตร์, ภาพถ่าย, TV เพราะมันให้ทั้ง 3 มิติ, shape, และ separation จาก background ในครั้งเดียว

3-Point Lighting Setup — มุมมองจากด้านบน

ตำแหน่งและความเข้ม

  • Key Light — Area Light ขนาดใหญ่ (~2m × 2m) วางมุม 45° ด้านหน้า-ซ้าย สูงกว่า subject เล็กน้อย — power 200–500W เป็น "หลัก"
  • Fill Light — Area Light ขนาดเดียวกัน ด้านหน้า-ขวา ตรงข้าม Key — power = 1/3 ของ Key (ลบเงาที่หนักเกิน)
  • Rim Light (Back Light) — Area Light ด้านหลังบนสุด ส่องลงมาที่ขอบ subject — power 60% ของ Key (สร้าง edge แยก subject จาก background)

ทำไม Area Light ดีกว่า Point Light

Point Light = แสงจากจุดเดียว → เงาแข็งคม (เหมือนแสงอาทิตย์ในทะเลทราย) Area Light = แสงจากพื้นที่ → เงานุ่ม (เหมือนแสงผ่านหน้าต่าง) — งาน MG ส่วนใหญ่ต้องการเงานุ่ม

ลองทำดู

scene ใหม่ + Cube + ลบ default light — เพิ่ม 3 Area Light ตามตำแหน่งและ power ในไดอะแกรม — render เปรียบเทียบกับ scene ที่มีแค่ light เดียว — เห็นความแตกต่างของ shape และ depth ทันที

HDRI & World Lighting

— environment ที่สมจริงทันที

HDRI = High Dynamic Range Image — รูป 360° ที่เก็บข้อมูลแสงทุกจุดในสภาพแวดล้อม ใช้แทน lighting setup ทั้งหมดในครั้งเดียว

ดาวน์โหลดฟรี (CC0) จาก polyhaven.com/hdris — มีหลายร้อยตัว: studio, sunset, indoor, sci-fi

การใช้

  1. เปิด Shader Editor → เปลี่ยน mode ที่ header เป็น World
  2. Add → Texture → Environment Texture
  3. กด Open → เลือกไฟล์ .exr หรือ .hdr ที่ดาวน์โหลด
  4. connect Color → Background → Surface ของ World Output
  5. ปรับ Strength 0.5–2 ตามต้องการ

HDRI ให้อะไรบ้าง

  • Ambient light — แสงจากทุกทิศ — กำหนดสี mood
  • Reflections — โลหะและกระจกจะสะท้อน environment ที่จริง — สมจริงทันที
  • Background — เห็น environment ใน render ได้ (ปิดได้ใน Render Properties → Film → Transparent)
เคล็ดลับ

HDRI + Area Light 1 ดวงสำหรับ key — ดีกว่า 3-point setup ในหลายกรณี เพราะ HDRI ให้ ambient ที่ realistic ส่วน Area Light เพิ่ม shape — เร็วกว่าและสวยกว่า

โมดูลนี้คุณได้: ใช้ camera อย่างที่ photographer ใช้, animate camera ด้วย Follow Path, เซ็ต 3-point lighting แบบ studio, และใช้ HDRI สำหรับ environment lighting

เคล็ดลับสุดท้าย: ก่อน render ดู scene ใน Rendered viewport shading เสมอ — บางครั้ง preview viewport ทำให้คุณเข้าใจผิดเรื่อง lighting

Eevee Next vs Cycles — เลือกอย่างไร

— ไม่มี engine ที่ดีที่สุด มีแต่ engine ที่เหมาะกับงาน

Blender มี render engine 3 ตัว เลือกที่ Render Properties → Render Engine

Eevee Next (Blender 4.2+)

  • Real-time rasterization — ใช้เทคนิคของวิดีโอเกม
  • เร็วมาก — render frame ละ 1–10 วินาที
  • รองรับ raytracing บางส่วน (reflection, shadow) ใน 4.2+
  • เหมาะกับ 80% ของงาน MG — โดยเฉพาะงาน abstract, motion design, UI animation
  • จุดอ่อน: caustics ผ่านน้ำ/แก้ว, refraction ซับซ้อน, complex SSS

Cycles

  • Path-tracing เต็มสูตร — เลียนแบบฟิสิกส์ของแสงจริงทุกอนุภาค
  • ช้ากว่า 10–50 เท่า — frame ละ 1–10 นาที
  • ผลลัพธ์ photoreal — glass, water, skin ดูจริงระดับ commercial
  • เหมาะกับ product viz, photoreal renders, complex materials
  • ใน MG — ใช้เฉพาะ shot ที่ต้องการ realism สูง

Workbench

  • render พื้นฐานแบบ flat — ไม่มี lighting จริง
  • ใช้ทำ preview/draft ตอน animate
  • เร็วสุด
workflow แนะนำ

เริ่มที่ Eevee เสมอ — ทำให้เสร็จ ถ้าผลดีพอใจ จบ ถ้าต้อง realism มากกว่า ค่อยเปลี่ยนเป็น Cycles — material, light, camera ส่วนใหญ่ใช้ตัวเดียวกันได้

Output Settings — ตั้งให้ถูกตั้งแต่ต้น

— save เป็น sequence ไม่ใช่ video

กฎทอง: render เป็น image sequence ไม่ใช่ video format

Output Properties

  1. File Format = PNG (ทั่วไป) หรือ OpenEXR Multilayer (สำหรับ post ที่ลึก)
  2. Color = RGBA (ถ้าต้องการ alpha) หรือ RGB
  3. Color Depth = 16 (ไม่ใช่ 8) สำหรับ post-processing — ถ้าตั้ง 8 จะเห็น banding ใน gradient
  4. Compression = 15% สำหรับ PNG (cheap, fast)
  5. Output path = //renders/shot01_ — เครื่องหมาย // = relative path (ดีต่อ portability)

หลัง render เสร็จได้ไฟล์ PNG หรือ EXR เป็น sequence (shot01_0001.png, shot01_0002.png, ...) ใช้ Blender's Video Sequence Editor หรือ FFmpeg compile เป็น MP4

# FFmpeg command: PNG sequence → MP4 ffmpeg -framerate 24 -i shot01_%04d.png \ -c:v libx264 -crf 18 -pix_fmt yuv420p \ output.mp4
เหตุผลสำคัญ

ถ้า render เป็น MP4 ตรงๆ แล้ว Blender crash กลางทาง — ไฟล์เสียทั้งหมด ต้องเริ่มใหม่ ถ้า render เป็น sequence — ยังมีไฟล์ที่เสร็จแล้ว เปลี่ยน Frame Range เริ่มจาก frame ที่ค้าง resume ต่อได้

Compositor — เพิ่มชั้น polish ขั้นสุดท้าย

— effect ที่ทำใน render ไม่ได้

เปิด workspace Compositing → tick Use Nodes ที่ header — node graph เปิดมามี 2 node: Render Layers, Composite

Compositor คือ "post-processing" หลัง render — เพิ่ม glow, lens flare, color tweak โดยไม่ต้อง render ใหม่

Effect ที่ใช้บ่อย

Render Layers ↓ Glare (Fog Glow, threshold 1.0) ↓ Lens Distortion (dispersion 0.005) ↓ Color Balance (lift / gamma / gain) ↓ Mix + Noise Texture (mode: Add, mix 0.04) ↓ Composite

แต่ละ node ทำอะไร

  • Glare — เพิ่ม bloom รอบจุดสว่าง — Fog Glow ให้ผลนุ่ม, Streaks ให้แสงพาด, Ghosts ให้ lens flare
  • Lens Distortion — distort เล็กน้อย + chromatic aberration — เพิ่ม "filmic" feel
  • Color Balance — color grade ตาม lift/gamma/gain (shadow/midtone/highlight)
  • Noise + Mix (Add) — film grain — mix 0.03–0.06
  • Vignette — ใช้ Ellipse Mask + Mix (Multiply) — มืดที่ขอบ focus ไปที่ center
เคล็ดลับ "filmic"

3 อย่างที่ทำให้ render ทุกตัวดู cinematic ทันที: Glare (Fog Glow weak) + Vignette (subtle) + Film Grain (mix 0.04) — ใช้คู่กันทุก project

Color Grading ด้วย AgX — workflow ปี 2026

— เลิกใช้ Filmic ได้แล้ว

Render Properties → Color Management — ตั้งค่าตามนี้:

  • Display Device = sRGB
  • View Transform = AgX
  • Look = Medium High Contrast (เริ่มต้น) — สามารถลองตัวอื่นได้
  • Exposure = 0 (default) ปรับได้ ±2
  • Gamma = 1.0

AgX vs Filmic

Filmic เป็น standard ของ Blender 2.x–3.x — ทำงานดีแต่มีปัญหากับ highlight สีอิ่มเข้มจะ "ขาวซีด" หรือ "เพี้ยนสี"

AgX (มาตรฐาน 4.x) — จัดการ highlight roll-off ดีกว่ามาก — ไฟ neon สีอิ่ม จะคงสีไว้ได้ skin tone ไม่เพี้ยน อิ่มสีเขียว/แดงไม่ "burn out"

FINAL POLISH

ก่อน export ครั้งสุดท้าย — เพิ่มใน compositor: Vignette อ่อนๆ + Grain + Sharpen 0.5 — 3 อย่างนี้ทำให้ render ทุกตัวดูเสมือน "filmic" เก่า ไม่ใช่ "render ใหม่ๆ"

ลองทำดู

Render frame เดียวกัน 2 ครั้ง — ครั้งหนึ่งใช้ Filmic, อีกครั้งใช้ AgX — เปิดเปรียบเทียบ Tab ใน image viewer — สังเกตความต่างของ highlight (จุดสว่าง) และ saturation

โมดูลนี้คุณได้: เลือก engine ตามงาน, render เป็น sequence, ใช้ Compositor เพิ่ม polish, และตั้ง color management ด้วย AgX

ภาคผนวก — ศัพท์ที่ควรรู้

เปิดดูระหว่างอ่านได้เมื่อเจอคำที่ไม่คุ้น ครอบคลุมศัพท์เทคนิค Blender + ทั่วไปของ MG

Keyframe /keyframe/
snapshot ของ property หนึ่ง ณ เวลาหนึ่ง — บอก Blender ว่า "ที่ frame X ค่าควรเป็น Y" ระหว่าง keyframe Blender จะคำนวณค่าระหว่างกลางให้เอง (interpolation)
F-Curve /function curve/
เส้นโค้งที่แสดง property ตามแกนเวลา ใน Graph Editor — แกน X = frame, Y = value แต่ละ keyframe คือจุดบนเส้น handle กำหนดความนุ่มของ curve
Easing /easing/
รูปทรงของ curve ระหว่าง keyframe กำหนดว่า value เปลี่ยน "เร็ว-ช้า" ยังไงระหว่างทาง Ease Out = เร็วก่อน ช้าจบ — ใช้บ่อยที่สุดใน MG
Driver /driver/
"keyframe ที่เป็นสมการ" — ผูก property ของ object ให้ขึ้นกับสมการที่อ้างอิงค่าของ object อื่น เช่น "ล้อหมุนตามตำแหน่ง X ของรถ"
Constraint /constraint/
กฎที่บังคับให้ object ทำตาม target — เช่น Track To = "หันหน้าหา target ตลอด", Follow Path = "เดินตาม curve"
Geometry Nodes /GN/
ระบบสร้าง geometry แบบ procedural ผ่าน node graph แทน keyframe ปรับ input ครั้งเดียว ทุกอย่างเปลี่ยนตาม
PBR /Physically Based Rendering/
มาตรฐาน shader ที่ใช้ฟิสิกส์จริงเลียนแบบวัสดุ — Principled BSDF ใน Blender ใช้ระบบนี้
HDRI /High Dynamic Range Image/
รูป 360° ที่เก็บข้อมูลแสง ใช้แทน lighting setup ในครั้งเดียว — ให้ ambient + reflection ที่ realistic
Compositor /compositor/
post-processing ภายใน Blender — ใส่ glow, color grade, grain, lens distortion โดยไม่ต้อง render ใหม่
AgX /agx/
color science ใหม่ใน Blender 4.x — จัดการ highlight roll-off ดีกว่า Filmic เดิม ทำให้สีอิ่มและ skin tone สวยขึ้น
Eevee Next /eevee/
render engine real-time ของ Blender ใช้เทคนิคของวิดีโอเกม + raytracing บางส่วน เร็วมาก เหมาะกับ MG ทั่วไป
Cycles /cycles/
render engine แบบ path-tracing เลียนแบบฟิสิกส์ของแสงทุกอนุภาค ช้าแต่ photoreal — เหมาะกับ product viz
